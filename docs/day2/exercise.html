Exercise: Linear systems in R and solution spaces

Goal

After this exercise you will be able to:

  • Build a coefficient matrix \(A\) and right-hand side \(\mathbf{b}\) in R and solve \(A\mathbf{x} = \mathbf{b}\) with solve.
  • Verify a solution with matrix multiplication %*%.
  • Recognise, by hand and in R, which of the three solution spaces a system has — one, none, or infinitely many — and tie that to whether a deconvolution problem is exactly-, over-, or under-determined.
  • Find a best-fit answer for an over-determined system with qr.solve / lm.

Background

A bulk expression sample is a mixture of cell types. With known signatures (columns of \(A\)) and a measured bulk profile \(\mathbf{b}\), the cell-type amounts \(\mathbf{x}\) satisfy \[ A\mathbf{x} = \mathbf{b}. \]

The shape of \(A\) tells you a lot. Let \(A\) have \(m\) rows (genes / equations) and \(n\) columns (cell types / unknowns):

  • \(m = n\), \(A\) non-singular \(\to\) exactly one solution (well-determined).
  • \(m > n\), equations conflict \(\to\) no exact solution (over-determined). Use least squares for the closest fit.
  • \(m < n\), or \(A\) singular \(\to\) infinitely many solutions (under-determined).

Two key facts:

  • \(A\) (square) is singular when one equation is a combination of the others — it carries no new information. Then \(\det A = 0\).
  • The rank of \(A\) is the number of genuinely independent equations. Full rank (\(=n\)) with a square \(A\) is exactly the unique-solution case.

By hand

Decide the solution space before running any R. For a \(2\times 2\) system, each equation is a line in the \((x_1, x_2)\) plane; the solutions are where the lines meet.

Problem 1 — classify three systems

For each, state whether it has one, none, or infinitely many solutions, and explain in one sentence (lines crossing / parallel / identical).

  1. \(\;x_1 + x_2 = 4,\quad x_1 - x_2 = 0\)

  2. \(\;x_1 + x_2 = 4,\quad 2x_1 + 2x_2 = 7\)

  3. \(\;x_1 + x_2 = 4,\quad 3x_1 + 3x_2 = 12\)

Problem 2 — which deconvolution scenario?

Match each situation to one solution / no solution / infinitely many, and to well- / over- / under-determined:

  1. You measure 5 marker genes to resolve 3 cell types, data is exact and consistent.
  2. You measure 2 marker genes to resolve 3 cell types.
  3. You measure 3 marker genes to resolve 3 cell types, signatures all distinct and independent.

In R

Step 1 — solve a well-determined system

A <- matrix(c(2, 1,
              1, 3), nrow = 2, byrow = TRUE)
b <- c(8, 9)

x <- solve(A, b)
x
# expected: 3 2

A %*% x          # should reproduce b = c(8, 9)

Step 2 — confirm your by-hand classifications

Run each system from Problem 1 through solve and read the result or the error message.

# (a) one solution
solve(matrix(c(1, 1, 1, -1), 2, byrow = TRUE), c(4, 0))

# (b) no solution (inconsistent, singular A)
solve(matrix(c(1, 1, 2, 2), 2, byrow = TRUE), c(4, 7))
# Error: ... system is exactly singular ...

# (c) infinitely many (consistent, singular A)
solve(matrix(c(1, 1, 3, 3), 2, byrow = TRUE), c(4, 12))
# Error: ... system is exactly singular ...

Note that solve gives the same “singular” error for (b) and (c): it cannot tell “no solution” from “infinitely many”. Use the diagnostics below to distinguish them.

Step 3 — diagnose without solving

A_bc <- matrix(c(1, 1, 2, 2), nrow = 2, byrow = TRUE)
det(A_bc)        # ~0  -> singular, so NOT a unique-solution system
qr(A_bc)$rank    # 1   -> only one independent equation (of 2 unknowns)

Use det, qr$rank, and dim to characterise a system before trying to solve it.

Step 4 — an over-determined system and least squares

Five genes, two cell types, noisy data: no exact solution, so find the best fit.

A <- matrix(c(2, 1,
              1, 3,
              1, 1,
              3, 2,
              0, 2), nrow = 5, byrow = TRUE)   # 5 genes, 2 cell types
b <- c(8.1, 8.9, 5.2, 10.8, 4.1)               # noisy measurements

# solve(A, b) would error: A is not square. Use least squares:
xhat <- qr.solve(A, b)
xhat

# Same answer via linear regression (no intercept):
coef(lm(b ~ A - 1))

# How far off is each equation? (the residuals)
A %*% xhat - b

We return to least squares and why it is the right notion of “closest” when we cover projection (Day 3) and regression later.

Hints

Hint 1 — building the matrix

Use matrix with byrow = TRUE so the numbers read in the same row order as the equations. Print the matrix and check it against the equations before solving.

Hint 2 — reading the lines (Problem 1)

Two lines either cross once (one solution), are parallel and distinct (no solution), or are the same line (infinitely many). Compare the coefficient ratios to the right-hand-side ratio: e.g. in (b) the left sides differ by a factor 2 but the right sides (\(4\) vs \(7\)) do not.

Hint 3 — matching scenarios (Problem 2)

Count \(m\) = number of marker genes (equations / rows) and \(n\) = number of cell types (unknowns / columns). Then use the shape rule: \(m > n\) is over-determined, \(m < n\) is under-determined, \(m = n\) with independent signatures is well-determined. “Exact and consistent” data means the extra equations agree, so an over-determined-by-shape system can still land on one exact solution; with noise it would have none and need least squares.

Hint 4 — what solve’s error means

solve errors with “system is exactly singular” for both no-solution and infinitely-many cases, and with “‘a’ must be a square matrix” for any non-square \(A\). The error tells you the system is not well-determined, but not which of the two singular cases it is.

Hint 5 — distinguishing none vs infinitely many

Both have singular \(A\). Check consistency: does \(\mathbf{b}\) lie in the column space? A practical check for these small cases is whether the proportionality of the rows of \(A\) is also matched by \(\mathbf{b}\). In Problem 1, (b) breaks it (\(4 \to 7\) is not \(\times 2\)) so no solution; (c) preserves it (\(4 \to 12\) is \(\times 3\)) so infinitely many.

Hint 6 — least squares functions

For an over-determined (tall, non-square) \(A\), use qr.solve(A, b) or fit a no-intercept regression with lm(b ~ A - 1) and read coef. The - 1 drops the intercept so the model is purely \(A\mathbf{x}\).

Solutions

Solution

Problem 1

  1. One solution. Adding the equations: \(2x_1 = 4 \Rightarrow x_1 = 2\), then \(x_2 = 2\). The two lines cross once. (Solution \(x_1 = x_2 = 2\).)

  2. No solution. Left sides are proportional (\(2\times\)) but right sides are not (\(4 \times 2 = 8 \ne 7\)). The lines are parallel and distinct — they never meet.

  3. Infinitely many. The second equation is exactly \(3\times\) the first (including the right side: \(3 \times 4 = 12\)). It is the same line; every point on \(x_1 + x_2 = 4\) is a solution.

Problem 2

  1. 5 genes, 3 cell types, consistent \(\to\) over-determined by shape but consistent, so it happens to have one solution here (with exact data the extra equations agree). With noise it would have no exact solution and need least squares.

  2. 2 genes, 3 cell types \(\to\) under-determined, infinitely many solutions — too few markers to resolve the cell types.

  3. 3 genes, 3 cell types, independent \(\to\) well-determined, one solution.

R, step by step

Step 1 — the well-determined system. First confirm the answer by hand so you know what to expect. The system is \(2x_1 + x_2 = 8\) and \(x_1 + 3x_2 = 9\). From equation 1, \(x_2 = 8 - 2x_1\); substitute into equation 2: \[ x_1 + 3(8 - 2x_1) = 9 \;\Rightarrow\; x_1 + 24 - 6x_1 = 9 \;\Rightarrow\; -5x_1 = -15 \;\Rightarrow\; x_1 = 3, \] then \(x_2 = 8 - 2(3) = 2\). So \(\mathbf{x} = (3, 2)\), which is what solve returns.

A <- matrix(c(2, 1, 1, 3), nrow = 2, byrow = TRUE)
solve(A, c(8, 9))           # 3 2
A %*% c(3, 2)               # 8 9  (check: reproduces b)

Step 2 — confirm the classifications. (a) is the one-solution system; adding the two equations gives \(2x_1 = 4\), so \(x_1 = 2\) and \(x_2 = 2\) — solve returns 2 2. (b) and (c) both have a singular \(A\), so solve throws the same “system is exactly singular” error for both and cannot tell them apart.

solve(matrix(c(1, 1, 1, -1), 2, byrow = TRUE), c(4, 0))   # 2 2
# (b) and (c) both throw "system is exactly singular"

Step 3 — diagnostics. For the (b)/(c) matrix, row 2 is \(2\times\) row 1, so the determinant is \(1\cdot2 - 1\cdot2 = 0\) and only one of the two rows is independent (rank 1). Rank \(1 < 2\) unknowns confirms the system is not unique without even attempting to solve it.

A_bc <- matrix(c(1, 1, 2, 2), nrow = 2, byrow = TRUE)
det(A_bc)        # 0
qr(A_bc)$rank    # 1   (< 2 unknowns -> not unique)

Step 4 — over-determined system, least squares. \(A\) is \(5\times 2\) (tall), so solve(A, b) would error (“‘a’ must be a square matrix”). qr.solve returns the least-squares estimate — the \(\mathbf{x}\) minimising \(\|A\mathbf{x}-\mathbf{b}\|^2\) — and lm(b ~ A - 1) gives the same numbers (the - 1 removes the intercept). The residual vector \(A\hat{\mathbf{x}}- \mathbf{b}\) is not all zeros: that is the signature of an over-determined system — no exact fit exists, and least squares returns the closest compromise.

A5 <- matrix(c(2,1, 1,3, 1,1, 3,2, 0,2), nrow = 5, byrow = TRUE)
b5 <- c(8.1, 8.9, 5.2, 10.8, 4.1)

qr.solve(A5, b5)            # 2.508511 2.089362
coef(lm(b5 ~ A5 - 1))       # A51 2.508511   A52 2.089362  (same values)

A5 %*% qr.solve(A5, b5) - b5
# row residuals, not zero:
# -0.9936170 -0.1234043 -0.6021277  0.9042553  0.0787234
